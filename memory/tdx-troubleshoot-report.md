## 三、故障根因分析

### 3.1 指数K线不可用
- `get_index_bars(category, market, code)` 在所有服务器上返回None
- **根因：** pytdx库的`get_index_bars`实现可能与当前TDX协议版本不兼容
- **影响：** 无法通过TDX获取上证指数、深证成指等指数的历史K线

### 3.2 证券列表问题
- 深圳返回的是分类代码（395001=主板A股、395002=主板B股等），不是个股列表
- 上海返回None
- **根因：** `get_security_list`接口返回的是市场分类，不是完整的证券列表
- **影响：** 无法通过TDX获取完整的股票代码列表

### 3.3 指数代码映射
- 上证指数在TDX中market=1, code=1，但`get_security_quotes`返回的是平安银行(000001)的数据
- **根因：** 指数和个股使用不同的代码体系，pytdx默认映射到个股

## 四、与AKShare的分工方案

| 功能 | 推荐工具 | 原因 |
|------|---------|------|
| 盘中实时报价 | TDX ✅ | 低延迟、高频轮询友好 |
| 盘中分时数据 | TDX ✅ | 240点完整分时 |
| 盘中预警监控 | TDX ✅ | 实时行情+分时，适合15分钟级监控 |
| 日K线/历史数据 | AKShare ✅ | 数据完整稳定 |
| 指数K线 | AKShare ✅ | TDX的get_index_bars不可用 |
| 申万行业数据 | AKShare ✅ | TDX无此功能 |
| 收盘后鱼盆计算 | AKShare ✅ | 数据清洗好，直接pandas处理 |

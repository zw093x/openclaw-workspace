| | `127.0.0.1:18789` | `42.193.183.176:30358` |
|------|------|------|
| **名称** | OpenClaw Web UI | OpenClaw Gateway |
| **位置** | 你本地 PC | 云服务器 |

- **Gateway (30358)** → 幕后干活的，负责接收飞书消息、触发定时任务、把回复发回飞书

所以你之前截图里的会话列表，就是从 Web UI 里看到的。而 Gateway 才是实际运行在云端、24小时服务的那个。

彭煜: ssh秘钥与root密码的区别

[20:18] 🤖 小P
[[reply_to_current]] **SSH 密钥 vs Root 密码**

| | SSH 密钥 | Root 密码 |
|---|---------|----------|
| **原理** | 非对称加密（公钥+私钥） | 对称认证（密码匹配） |
| **安全性** | 🔒 极高 — 私钥不传输，无法暴力破解 | ⚠️ 中等 — 可被暴力破解/字典攻击 |
| **便利性** | ✅ 免密登录，一条命令搞定 | ❌ 每次要输入密码 |
| **配置** | 需要提前部署公钥到服务器 | 不需要额外配置 |
| **适用场景** | 日常运维、自动化脚本 | 紧急救援、临时访问 |

**类比：**
- **密码** → 像开门的钥匙，谁拿到都能开
- **密钥** → 像指纹锁，只有你的私钥能配对

**建议：** 配置好 SSH 密钥后，禁用密码登录，安全性大幅提升。

需要我帮你配置 SSH 密钥吗？

[[reply_to_current]] **SSH config 是什么：**

就是你本地 PC 上的一个配置文件（`~/.ssh/config`），用来**简化 SSH 命令**。

**没有 config 时，你每次要打：**
```bash
ssh -R 18789:127.0.0.1:18789 root@42.193.183.176

ssh myserver

甚至隧道也可以自动建立。

```
Host myserver
    HostName 42.193.183.176
    User root
    IdentityFile ~/.ssh/id_ed25519
    RemoteForward 18789 127.0.0.1:18789
```

放在你本地 PC 的 `~/.ssh/config` 文件里。

    HostName 42.193.183.176
    User root
    IdentityFile ~/.ssh/id_ed25519
    RemoteForward 18789 127.0.0.1:18789

chmod 600 ~/.ssh/config
```

之后只需要 `ssh myserver` 就能登录+建立隧道了。

## 自愈系统架构

```
scripts/self_heal.py          ← 核心引擎
scripts/self_heal_registry.json ← 故障记忆库
scripts/self_heal_log.jsonl     ← 运行日志
```

## 功能

| 参数 | 功能 |
|------|------|
| `--report` | 扫描 + 生成健康报告（只读） |
| `--fix` | 扫描 + 自动修复 |
| `--learn` | 扫描 + 学习新错误模式 |
| `--fix --learn` | 全功能：扫描 + 修复 + 学习 |

## 覆盖的故障类型

| 故障 | 自动修复方式 |
|------|-------------|
| cron delivery 缺失 | 补全 channel + to，重启 gateway |
| cron task error | delivery 修复后自动恢复 |
| 飞书连接断开 | 重启 gateway |
| gateway 不可达 | 重启 gateway |
| 磁盘空间不足 | 清理日志和临时文件 |
| 内存过高 | 告警（无法自动修复） |

## 自动化

| 机制 | 频率 | 作用 |
|------|------|------|
| Cron 自愈系统 | 每小时 | 自动扫描 + 修复 + 学习 |
| 飞书连接监控 | 每30分钟 | 专项检测飞书断联 |
| Cron 自检修复 | 每6小时 | 专项检查 delivery 配置 |
| 故障库 | 持续积累 | 记录所有错误模式，越用越聪明 |

**核心理念：每次故障只发生一次 → 记录 → 自动修复 → 永久预防** 🐾

sessions.json 是什么

[13:55] 🤖 小P
[[reply_to_current]] sessions.json 是 OpenClaw 的**会话列表索引文件**，记录在：

```
~/.openclaw/sessions.json
```

**作用：**
- 记录所有会话的元信息（sessionKey、创建时间、最近活动时间、使用的模型等）
- 供 `sessions_list` / `sessions_history` 等工具查询和读取历史会话

**简单理解：** 就是"所有聊天记录的清单"，方便定位历史对话。

⚠️ 不要手动编辑，OpenClaw 自动维护。

[20:16] 🤖 小P
[[reply_to_current]] **OOM = Out of Memory（内存耗尽）**

当进程使用的内存超过服务器可用内存时，Linux 内核会主动杀死（kill）最耗内存的进程来保护系统，这叫 **OOM Killer**。

**在你的场景：**
- 云服务器内存有限（约 4GB）
- Gateway 已经用了约 1.3GB
- 重启时需要重新加载所有插件和会话数据
- memory-tdai 插件额外占用不少内存
- 总需求超过可用内存 → OOM Killer 出手 → 进程被 SIGKILL

**简单说：内存不够用，系统强制关进程。**

这就是为什么之前 sessions.json 63MB 时 Gateway 崩溃了——内存不够用。

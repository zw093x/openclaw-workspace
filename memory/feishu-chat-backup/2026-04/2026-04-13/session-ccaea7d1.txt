# 飞书插件问题排查

## ❓ 常见问题（FAQ）

### 卡片按钮点击无反应

**现象**：点击卡片按钮后没有任何反应，然后提示报错.

**原因**：应用未开通「消息卡片回传交互」权限。

**解决步骤**：

1. 登录飞书开放平台：https://open.feishu.cn/app
2. 选择您的应用 → **事件与回调**
3. 在回调配置中,修改订阅方式为"长链接"并添加回调 "卡片回传交互"(card.action.trigger)
4. 创建应用版本 → 提交审核 → 发布

## 🐧 Ubuntu 系统 vs 🐳 Docker/Root 系统部署 OpenClaw

### Ubuntu 系统（直接安装在 Host）

| 优势 | 劣势 |
|------|------|
| ✅ **性能零损耗** — 无容器开销 | ❌ **污染系统环境** — 全局安装 Node/npm |
| ✅ **配置简单** — 无端口映射烦恼 | ❌ **升级风险高** — 影响现有服务 |
| ✅ **访问全部系统资源**（GPU/网络/文件系统） | ❌ **权限风险大** — 漏洞可接管整台机器 |
| ✅ **调试直观** — 直接看进程/日志 | ❌ **迁移困难** — 依赖系统环境 |

### Docker/Root 系统（容器化）

| 优势 | 劣势 |
|------|------|
| ✅ **环境隔离** — 不影响主机 | ❌ **性能损耗** — 5-15% 资源开销 |
| ✅ **一键部署/迁移** — 环境一致 | ❌ **GPU直通复杂** — 需要 --gpus 选项 |
| ✅ **权限最小化** — 漏洞不蔓延 | ❌ **调试不便** — 得进容器看日志 |
| ✅ **版本独立** — 多版本共存 | ❌ **端口/网络配置繁琐** |

## ❓ 常见问题（FAQ）

### 卡片按钮点击无反应

**现象**：点击卡片按钮后没有任何反应，然后提示报错.

**原因**：应用未开通「消息卡片回传交互」权限。

**解决步骤**：

1. 登录飞书开放平台：https://open.feishu.cn/app
2. 选择您的应用 → **事件与回调**
3. 在回调配置中,修改订阅方式为"长链接"并添加回调 "卡片回传交互"(card.action.trigger)
4. 创建应用版本 → 提交审核 → 发布

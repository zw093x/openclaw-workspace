## 08:11 - 11:06 Cron 投递故障修复 + 推送体验优化
### 1. 批量修复 cron 投递失败（10个任务）
**根因（3类错误）：**
- `Channel is required when multiple channels are configured` — 配置 lightclawbot+飞书双渠道后 cron delivery channel 解析失败（6个任务，从3/28开始存在）
- `Delivering to Feishu requires target` — 飞书投递目标格式问题（2个任务）
- `⚠️ ✉️ Message failed` — 瞬态投递竞态，消息实际已送达（每日早报，1个）
**修复：** 批量重新设置 delivery 参数 + 手动触发验证
### 2. 飞书卡片投递系统
**背景：** 用户希望日报/资讯推送以飞书卡片形式展示（而非纯文字）
**技术方案：**
- 创建 `scripts/feishu_card_send.py` — 直接调用飞书 API 发送 interactive card（不依赖 OpenClaw cron agent 调用 message 工具）
- 创建 `scripts/cron_card_wrapper.py` — 包装器：触发 cron 任务 → 等待完成 → 获取内容 → 调用飞书 API 发卡片
- **关键发现：mimo-v2-pro 在 cron 隔离会话中不调用任何工具**（exec/read/web_search 均不调用）
**已迁移为卡片投递的任务（18个）：**
- 通过系统 crontab 调度 + cron_card_wrapper.py 投递
- OpenClaw cron 设置 `--no-deliver`（禁用文本投递，避免重复）
### 3. 天气数据预注入方案
**问题：** cron agent 不调用工具，无法获取实时天气
**方案：**
- 创建 `scripts/get_weather.py` — 获取逐时天气预报（wttr.in API）
- 创建 `scripts/weather_preload.py` — 预加载天气数据，注入到 cron prompt 中
- 系统 crontab 每天 07:25/07:55 自动运行预加载

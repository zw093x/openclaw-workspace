**版本：v1.0 | 基于扩散模型技术原理（DDPM/DDIM/SDXL/Flux）+ 2026年最新实践**
**适用平台：Stable Diffusion WebUI (A1111/Forge) / ComfyUI v0.3+**

## 一、绘画万能框架深度拆解

### 1.1 框架总公式

```
[核心主体：主体词 + 核心特征(特征强化词)]
+ [场景环境：背景 + 氛围]
+ [构图视角：景别 + 角度 + 焦距]
+ [光影色调：光源 + 光质 + 色调 + 影调]
+ [视觉风格：艺术家/流派/材质]
+ [画质标准：分辨率 + 画质词 + 渲染引擎]
+ [约束规则：负面提示 + 局部约束]
```

### 1.2 各模块权重贡献比

| 模块 | 权重贡献比 | 调整幅度 | 典型范围 |
|------|-----------|---------|---------|
| 核心主体 + 特征 | 35% | ±15% | 主体词必须精确，特征词决定相似度 |
| 视觉风格 | 20% | ±10% | 风格词决定60%以上视觉走向 |
| 光影色调 | 18% | ±8% | 影调决定情绪基调 |
| 构图视角 | 12% | ±5% | 构图决定叙事结构 |
| 场景环境 | 10% | ±8% | 背景可模糊，氛围影响代入感 |
| 画质标准 | 3% | ±2% | 收尾用，过高反而引入伪影 |
| 约束规则 | 2% | — | 宁少勿多 |

### 1.3 模块底层原理与典型取值

#### 模块1：核心主体（主体词 + 核心特征）
- **底层原理**：CLIP embedding决定语义锚点位置；核心特征通过细粒度特征增强修正高层语义
- **权重逻辑**：主体词默认1.0，偏离→提高至1.1~1.3；核心特征（specific attribute）→1.1~1.2

#### 模块2：场景环境（背景 + 氛围）
- **底层原理**：背景embedding与主体embedding在UNet中竞争注意力分配，背景越详细稀释越严重
- **权重逻辑**：背景精确度需求低→0.5~0.8；氛围词→0.8~1.0
- **边界条件**：背景词>5个时，稀释效应显著增加

#### 模块3：构图视角（景别 + 角度 + 焦距）
- **景别心理效应**：大特写→亲密/紧张；特写→强调表情；中景→叙事感；全景→史诗感；大远景→渺小感
- **视角心理效应**：仰视→威严/压迫；俯视→弱势/怜悯；平视→平等/真实

#### 模块4：光影色调
- **光源类型**：自然光(golden hour)、伦勃朗光(dramatic)、蝴蝶光(glamour)、平面光(clean)、边缘光(ethereal)、逆光(romantic/silhouette)
- **色调**：暖色调→温馨/怀旧；冷色调→科技/冷静；莫兰迪→高级/文艺；赛博朋克→科幻/都市颓废

#### 模块5：视觉风格
- **底层原理**：Style Embedding——模型在训练中学到的艺术风格统计特征，像素分布规律差异

#### 模块6：画质标准
- **底层原理**：画质词通过高频特征增强在潜空间施加额外梯度，SDXL/Flux中作用已弱化，过度使用反而引入伪影

## 二、核心技术模块深度研究

### 模块A：结构化提示词与权重精细化控制

#### A.1 CLIP编码器的文本-视觉对齐机制

SD系列使用OpenCLIP（SD1.x=CLIP ViT-L/14，SDXL=OpenCLIP ViT-g/14，Flux=CLIP ViT-L+TiMi双编码器）

```
Token序列 → Tokenizer → Token IDs (max 77 tokens)
         → CLIP Text Encoder → 768-dim (SD1.x) / 1280-dim (SDXL) / 768+768 (Flux)
         → 77 × embedding_dim 矩阵
         → 交叉注意力层注入UNet
```

**关键机制：**
- 前20个token注意力更强，核心主体词应置于前1/3
- BPE编码："cyberpunk" ≠ "cyber"+"punk"（分词方式不同），单独写更稳定
- 后置词（50-77位）容易被截断，风格词放中段，画质词放末尾

#### A.2 权重边界效应（SD1.5/SDXL/Flux对比）

| 权重范围 | 行为 | 副作用 |
|---------|------|--------|
| 0.1~0.4 | 极弱影响，可忽略 | 无 |
| 0.5~0.8 | 轻微影响，可被覆盖 | 适合背景元素 |
| 0.9~1.1 | 标准影响，基准 | 基准范围 |
| **1.2~1.4** | 明显强化 | **推荐区间** |
| 1.5~1.8 | 强烈强化 | 形态扭曲/过饱和 |
| 1.9~2.5 | 极强 | 高概率碎片化 |
| >3.0 | 极值 | 模型崩溃，输出噪声 |

- **SDXL**：对高权重(>1.5)耐受度更高（CLIP维度1280d，嵌入空间更稀疏）
- **Flux**：权重敏感度更高，>1.4即有明显副作用，推荐上限**1.3**

#### A.3 权重选择决策

| 场景 | 推荐权重 |
|------|---------|
| 主体身份标识（人种、发色） | **1.2~1.3** |
| 装饰属性（帽子、饰品） | **1.1~1.2** |
| 风格词/光影词 | **1.2~1.3** |
| SD1.5主体词 | 上限**1.4** |
| Flux专用 | 上限**1.3** |

#### A.4 SD WebUI语法

```
(prompt:1.5)         # 整体加权
((prompt))           # 双重加权 1.1×1.1=1.21
prompt::0.9          # 递减采样（步数递减时淡出）
[prompt]             # 降低权重（部分fork支持）

# Composable Diffusion 递减技巧：
full body shot::0.95, dynamic pose::0.9, action::0.8
# 动态姿势在前30%步数主导，后逐渐弱化
```

### 模块D：可控生成配套技术

#### D.1 LoRA权重经验值

| LoRA类型 | strength_model | strength_clip | 说明 |
|---------|----------------|--------------|------|
| 角色LoRA | 0.6~0.8 | 0.8~1.0 | 角色身份靠CLIP激活 |
| 风格LoRA | 0.7~1.0 | 0.5~0.8 | 风格主要影响UNet |
| 概念LoRA | 0.5~0.7 | 0.7~0.9 | 平衡型 |
| 服饰LoRA | 0.4~0.6 | 1.0~1.2 | CLIP权重略高 |

**ComfyUI LoRA节点配置：**
```
节点：LoRA Loader
参数：
  - name = filename.safetensors
  - strength_model = 0.7（UNet影响强度）
  - strength_clip = 0.8（CLIP影响强度）
```

#### D.2 ControlNet各模式适用场景

| 模式 | 适用场景 | 控制精度 | 副作用 |
|------|---------|---------|-------|
| **Canny（硬边缘）** | 线稿上色、建筑、产品 | 极高 | 过度依赖边缘，可能僵硬 |
| **SoftEdge/HED** | 人物轮廓，写实场景 | 高 | 较自然，略松弛 |
| **Depth（深度图）** | 建筑、场景、构图控制 | 高 | 深度估计误差会传递 |
| **Normal Map** | 表面凹凸、建筑 | 中高 | 对纹理敏感 |
| **Pose（姿态）** | 人物姿态、动作 | 中 | 手部/脚部误差大 |
| **Seg（语义分割）** | 区域内容控制 | 高 | 分割精度依赖 |

#### D.3 提示词 + ControlNet协同策略

| 任务 | 提示词负责 | ControlNet负责 |
|------|-----------|--------------|
| 建筑线稿上色 | 色调、氛围、材质 | 线稿结构（Canny）|
| 人物精准姿态 | 主体特征、风格，光影 | 姿态（Pose/OpenPose）|
| 场景构图控制 | 风格，光影、天气 | 深度/法线（Depth/Normal）|

## 三、实战对照实验设计

**实验：验证"光影色调"与"视觉风格"的交互权重对主体还原度的影响**

| 实验组 | 主体权重 | 光影权重 | 风格权重 | 预期结果 |
|--------|---------|---------|---------|---------|
| A（对照） | 1.0 | 1.0 | 1.0 | 基线 |
| B（光影↑） | 1.0 | 1.5 | 1.0 | 氛围增强，主体可能弱化 |
| C（风格↑） | 1.0 | 1.0 | 1.5 | 风格明显，主体可能被艺术化稀释 |
| D（双高） | 1.0 | 1.3 | 1.3 | 平衡强化 |
| E（主体保护） | 1.3 | 1.0 | 1.0 | 主体突出，背景简洁 |

**量化评估指标（禁止纯主观评价）：**
- **CLIP-I**：图像-图像相似度
- **CLIP-T**：文本-图像匹配度
- **BLIP-2主体检测**：主体识别准确率
- **FID Score**：需准备参考图集

## 四、高阶优化技术

### 主体一致性高级技术对比

| 技术 | 原理 | 适用场景 | 副作用 |
|------|------|---------|-------|
| **IP-Adapter** | 解耦图像提示与文本编码器 | 参考图控制主体 | 需配合提示词 |
| **Reference-Only** | 直接将参考图特征注入潜空间 | 快速主体迁移 | 可能污染整体风格 |
| **InstantID/FaceID** | 人脸特征解耦+注意力注入 | 精准人脸一致性 | 依赖检测准确性 |
| **LoRA（训练型）** | 通过低秩矩阵微调注意力层 | 高度定制角色/风格 | 需训练，成本高 |

### APE自动化提示词优化流程

```
Step 1: 用大模型生成20组候选提示词（同一目标的变体）
Step 2: 批量生成（固定seed=42，固定参数）
Step 3: 用CLIP-T评分筛选Top5
Step 4: 对Top5做A/B测试（5个不同seed）
Step 5: 人工审核 → 提取共同特征 → 形成模板
```

## 结论总览

1. **框架优先级**：主体(35%) > 风格(20%) > 光影(18%) > 构图(12%) > 场景(10%) > 画质(3%)
2. **权重安全上限**：SD1.5上限1.4，SDXL/Flux上限1.3
3. **位置法则**：核心主体前置，画质词末尾，中段放风格
4. **ControlNet协同**：提示词主风格，ControlNet主结构
5. **LoRA权重**：角色用CLIP权重略高，风格用UNet权重略高
